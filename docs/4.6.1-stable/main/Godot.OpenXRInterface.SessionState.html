Enum OpenXRInterface.SessionState

GodotSharp.dll
public enum OpenXRInterface.SessionState : long

Fields

Exiting = 8

The OpenXR instance is about to be destroyed and we're exiting. InstanceExiting is emitted when we change to this state.

Focused = 5

The application has synched its frame loop with the runtime, we're rendering output to the user and we're receiving XR input. SessionFocussed is emitted when we change to this state.

Note: This is the state OpenXR will be in when the user can fully interact with your game.

Idle = 1

The initial state after the OpenXR session is created or after the session is destroyed.

LossPending = 7

The session is about to be lost. SessionLossPending is emitted when we change to this state.

Ready = 2

OpenXR is ready to begin our session. SessionBegun is emitted when we change to this state.

Stopping = 6

Our session is being stopped. SessionStopping is emitted when we change to this state.

Synchronized = 3

The application has synched its frame loop with the runtime but we're not rendering anything. SessionSynchronized is emitted when we change to this state.

Unknown = 0

The state of the session is unknown, we haven't tried setting up OpenXR yet.

Visible = 4

The application has synched its frame loop with the runtime and we're rendering output to the user, however we receive no user input. SessionVisible is emitted when we change to this state.

Note: This is the current state just before we get the focused state, whenever the user opens a system menu, switches to another application, or takes off their headset.
